首先要知道，DOM事件流（event flow）有三个阶段：事件捕获阶段，目标阶段（target），事件冒泡阶段。

事件捕获 Event capturing

假设有个三个元素，Element1， Element2， Element3，在HTML里面的结构如下所示：
如果三个元素都注册了点击事件（click），当你点击了Element3，事件会从Element1开始触发，接着Element2，最后是Element3。

事件冒泡 Event bubbling

但如果是事件冒泡，点击了Element3，事件最先会在Element3触发，然后是Element2，最后是Element1。

到底哪个事件先发生？

当你知道了事件捕获和事件冒泡之后，你可能会有疑问，如果这三个元素不仅有事件捕获，又有事件冒泡，那么事件的发生顺序是怎么样的呢？

总结一下：

1.在非目标元素上，先执行捕获，再执行冒泡。

2.在目标元素上，按照事件的注册顺序依次执行，不分捕获还是冒泡。
